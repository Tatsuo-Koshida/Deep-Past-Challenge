# Notebook naming rules (`notebooks/`)

このリポジトリでは、`notebooks/<NNN>/` 配下の `.ipynb` を **「同一のオリジナル（公開Notebook / 自作Notebook）」単位**でグルーピングし、コピー（子/孫/ひ孫…）も同じフォルダに集約する。

## 1) フォルダ (`notebooks/<NNN>/`) の意味

- 例: `notebooks/002/`
- 同一フォルダに置くもの:
  - **オリジナル（根）**のNotebook
  - そのオリジナルをコピーして改変したNotebook（子）
  - 子をさらにコピーして改変したNotebook（孫）
  - …以降の世代（ひ孫、玄孫…）も **すべて同一フォルダ**

## 2) オリジナル（根）の命名

- **先頭に `[]` を付けない**
- **末尾に `-v<数字>` を付けない**
- 例: `notebooks/002/dpc-starter-train-cv5.ipynb`

## 3) コピー系（子/孫/ひ孫…）の命名

コピーしたNotebookは、**先頭に `[<lineage_id>]`** を付け、**末尾に `-v<version>`** を付けて世代を区別する。

### 3.1 `[<lineage_id>]`（系統ID）

- 同一フォルダ内で、**同じオリジナル由来の “系統（lineage）”** を表す番号
- **子/孫/ひ孫…は同じ `[<lineage_id>]` を共有**する
- オリジナルから別ルートで新規にコピーを作る（並行ブランチ）場合は、**別の `[<lineage_id>]`** を割り当てる（どちらも `-v1` から開始してよい）
- 途中バージョンから枝分かれした場合は、**親の系統IDに「枝分かれ元バージョン」をハイフンで連結**して新しい系統IDにする
  - 例: `[3]...-v2` から枝分かれ → 新系統は `[3-2]...-v1`
  - さらに `[3-2]...-v4` から枝分かれ → 新系統は `[3-2-4]...-v1`
  - 表記は原則として ASCII の `-`（ハイフン）を使う（`−` などの別文字は使わない）

### 3.2 `-v<version>`（世代/コピー回数）

- 子: `-v1`
- 孫: `-v2`
- ひ孫: `-v3`
- …のように **コピーを重ねるごとにインクリメント**する

### 3.3 形式（テンプレ）

- `notebooks/<NNN>/[<lineage_id>]<base_name>-v<version>.ipynb`
  - `<base_name>` はオリジナルのファイル名（拡張子 `.ipynb` を除く）を基本にする

## 4) 具体例

オリジナル:

- `notebooks/002/dpc-starter-train-cv5.ipynb`

同一系統（`[3]`）の世代:

- `notebooks/002/[3]dpc-starter-train-cv5-v1.ipynb`（子）
- `notebooks/002/[3]dpc-starter-train-cv5-v2.ipynb`（孫）
- `notebooks/002/[3]dpc-starter-train-cv5-v3.ipynb`（ひ孫）

並行系統（別ブランチ）:

- `notebooks/002/[4]dpc-starter-train-cv5-v1.ipynb`（オリジナルから別ルートで作った子）

## 5) 命名チェックリスト（最低限）

- オリジナル: `[]` なし、`-v` なし
- コピー: `[<数字>]` あり、`-v<数字>` あり
- 世代が上がるほど `v` が増える（`v1` → `v2` → `v3` …）
- 同一系統は同じ `[<lineage_id>]` を維持する

## 6) コピー作成時の差分メモ（必須）

コピーNotebook（`-v<version>` 付き）を新規に作るときは、**冒頭（最初）のセル**に「元（コピー元）から何を変えたか」を **日本語で**必ず書く。

対象に含めるもの:

- **バージョン更新**（例: `[3]...-v1 → [3]...-v2`）
- **枝分かれ（新系統の作成）**（例: `[3-2]...-v4` から枝分かれ → 新系統 `[3-2-4]...-v1`）

- 目的: 後から見たときに、`v1 → v2` の差分と意図が即座に追える状態にする
- 形式: Markdownセル推奨（ただし最初のセルであることを優先）

テンプレ（例）:

- 対象: `[3]xxx-v2.ipynb`
- 元: `[3]xxx-v1.ipynb`
- 種別: バージョン更新 / 枝分かれ
- 変更点:
  - （例）前処理: 正規化ルールAを追加
  - （例）学習: `max_len=256 → 384`、`lr=2e-5 → 1e-5`
  - （例）推論/生成: `generation_num_beams: 2 → 3`（= 探索幅を増やす）
- 変更理由/仮説: （例）長文の欠落を減らしたい
- 実行メモ（任意）: seed / CV設定 / 主要コマンド など

補足:

- 「セル数」「変更セル数」などの **機械的な差分** だけではなく、**変更元から取り込んだアイデア/施策（何を狙って何を変えたか）** を優先して書く
